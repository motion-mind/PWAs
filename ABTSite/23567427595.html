Replacing a notification class template of a device

Existing devices must be updated with the modified notification class template.

The notification class template cannot be changed for the following device groups:
- Routers and gateways
- DXR1, DXR2, PXC3 devices
- All devices with *.A.

Notification classes

  1. Go to Building > Building structure.
  2. Open the Application & device overview task.
  3. Select a device in the building hierarchy.
    Note: Multi-selection of devices is supported. If more than 15 devices are present and the user applies notification class for all, then a progress dialog displays. The messages are shown in the Log tab.
  4. Right-click a device and select Apply notification classes.
  5. The Apply Notification class dialog box opens.
  6. Click Yes.
  1. The defined project notification class is assigned to the device.
    Note: If an unsupported device is selected, an error message will be displayed in the Log tab.
    Notification class template

The user has to manually update the notification class number for alarmable objects like event enrollment, trend log objects. Verify the alarm routing as well.
