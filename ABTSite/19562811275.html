Importing an alternative BACnet/SC root certificate

When creating a project, ABT Site creates a root certificate, which is valid for the whole project. You can also import an alternative root certificate in ABT Site. This may be the case e.g. when your IT department requires that you work with a proprietary root certificate, generated by a dedicated CA (certificate authority) chosen by the IT department.

In case you need to roll back to the internal root certificate of ABT Site you can regenerate the internal root certificate.
Regenerate BACnet/SC internal root certificate

 

Import an alternative root certificate

  1. You have an alternative root certificate e.g. from the IT department.
  1. Go to Settings.
  2. Open the Certificates task.
  3. Select the BACnet/SC certificates tab.
  4. In the toolbar, click Import external signing certificate.
  5. Select the root certificate, and click Open.
  6. You have imported a root certificate. The old root certificate is still present in the project for a transition time until you download to all devices.
  7. As soon as every node is using the new root certificate, you can delete the old (temporary) root certificate.
  8. Download the configuration to the devices to remove the temporary root certificate from the devices.
