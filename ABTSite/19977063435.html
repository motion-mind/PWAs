Customizing BACnet object instance numbers

After customizing BACnet object instance numbers, a full load of the respective devices is required.
After modifying an instance number, the existing BACnet clients, such as Desigo CC or Desigo Control Point must re-engineer again.

BACnet object instance numbers have a default value when the BACnet object is created. It is possible to modify the value of BACnet object instance numbers. You can export a .CSV file containing BACnet object instance numbers, modify values and import the .CSV file again.

Export .CSV file

  1. Data point is created and assigned.
    Creating I/O data points
  2. You are in Building > Building structure view.
  1. Select a device or a building element containing a device.
  2. Right-click and select Create export....
  3. The Create export dialog box opens.
  4. Select BACnet object instances.
  5. Click OK.
  6. An export container with a .CSV file is created and exported.

Customize BACnet object instance numbers

Only changes of values in column OBJECT INSTANCE NUMBER are recognized by ABT Site after import. All other changes will be ignored.

  1. An export container with a .CSV file is created and exported.
  1. Click Open folder.
  2. Unzip the export container and open the .CSV file with Microsoft Excel.
  3. In column OBJECT INSTANCE NUMBER, change the value as required and save the .CSV file.

Import .CSV file

  1. In the context menu of the selected device, select Import BACnet object instances....
  2. Select .CSV file.
  3. Click Open.
  4. .CSV file is imported to ABT Site and new values are saved.

ABT Site validates the uniqueness and format of BACnet object instance numbers during the import. Any incorrect values (e.g., duplicated BACnet object instance numbers or a wrong format) block the import. In this case it is required to correct the values and import the .CSV file again.
