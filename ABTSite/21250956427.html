[Dmp29] Damper, open/close function & MI for feedback

Program block, name / node subtype: [Dmp29] Damper, open/close function & MI for feedback

Library hierarchy: Ventilation and air conditioning equipment

Family: DmpShof

Project tree

Diagram

Configurator

Program blocks are stored in the library without I/Os. Use the auto-create and auto-connect functions to create I/Os and/or to connect them to the interface blocks, e.g., R_A, CMD_B. Alternatively, take the I/Os from the folder containing the predefined I/Os in the library and/or from the plant and connect them to the interface blocks.

Function

Controls a shutoff damper with open/close function and optional feedback signals and fault alarm.

The opening and closing time can be parameterized in the program.

This program block has the following optional functions:

Option: Fault (1xBI)

Reads a fault contact and generates an alarm.

This option is mainly used for dampers with actuators that are integrated over bus, e.g., Modbus.

Option: Feedback (1xMI=2xBI)

Reads the end switches of the actuator for the opened and closed position of the damper.

The program logic reads the binary command, considers the opening and the closing time, compares the signals from the end switches of the actuator from the option "Feedback" to the command and can generate two alarms (binary calculated values): "Position monitoring (Opened)" and "Position monitoring (Closed)". If "Position monitoring (Opened)" is in alarm, the air handling unit stops.
