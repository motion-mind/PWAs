New construction projects BACnet network structure example

In new construction projects, BACnet/SC-native and BACnet/SC-ready devices should be used as much as possible to make the transition to more secure BACnet networks easier for the customer.

Since BACnet/SC is only secure within the BACnet/SC logical network, any non-BACnet/SC networks must be secured using traditional methods. .

BACnet/SC renders BBMDs obsolete within the BACnet/SC logical network (hub and nodes). Even though BACnet/SC does not require BBMDs for nodes to communicate with the hub when going across IP subnets, BBMDs are still required for any existing BACnet/IP devices on different IP subnets in the project.

The DXR2.E are BACnet/SC-ready devices. Once their firmware is upgraded with BACnet/SC capability, the BBMD configurations can be removed and the whole BACnet Internetwork will be secure using BACnet/SC.

When designing mixed BACnet/SC and BACnet/IP networks in large projects with several or more IP subnets where BBMDs may be needed for the BACnet/IP portion of the network, careful configuration of BACnet routing and the BBMDs’ Broadcast Distribution Table must be ensured to avoid network loops or multiple paths to the same device.
