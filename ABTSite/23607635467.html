Reading and creating M-bus data points from online device data

Read and create data points from an online M-bus device.

  1. Before you add a M-bus network, make sure to configure and load the device.
    Configure and download.
  1. Go to Engineering.
  2. Open the M-bus task.
  3. Click Add network.
    • Select the M-bus (MBUS) port in the Port field and click OK.
  4. You have added an M-bus network.
  5. Click  to go online.
  6. Start the network scan.
    M-bus network scan.
  7. Right-click the device in the orphan list and select Read device data.
  8. The Reading device dialog box displays, where a trace of all the online device data points is read.
    • Select the Response T-out, Inter Character T-out and Inter Frame Wait Time in the respective fields and click Start. For more information on the time out characters, see Reading device - Time out characters.
  10. After a few seconds, the list of the data points in the online device displays.
  11. You can see all the discovered data points and ABT site will propose a BACnet object mapping.
  12. Select and tick/check the required data points and click Create.
    Note: Creating a customized device will force ABT Site to go offline automatically.
  13. The M-bus editor goes offline and the customized M-bus device is created, which displays under M-bus network.
  14. Click on the newly created M-bus device.
  15. The data points display below in the workplace table.
  16. Check the properties of the newly created M-bus device in the Properties tab.

ABT site can only propose an integration variant for the discovered data points. If further customization is required, the user should do it manually.

Any device with primary address 0, after device read out the Network address will change to 253 in the online mode.

For multiple devices with primary address 0, the uniqueness check is disabled.

Reading device - Time out characters

The read device time out characters define the time frame for the M-bus message to be transmitted. There are three characters time outs:

  • Inter character T-out: The entire M-bus message frame must be transmitted as a continuous stream of characters. The standardized inter-character timeout on the receiver is 22 bit times. Any gap > 22 bit times shall be considered as the end of a telegram.
  • Inter frame wait time: The minimum inter-frame wait time is 11 bit times between a received response frame and sending of the next request frame.
  • Response T-out: M-Bus standard defines the maximum acceptable delay time which leads to a response timeout in the manager. The standard delay time between the end of a manager telegram and the beginning of the subordinate response telegram shall be between 11 bit times and (11 + 330 bit times + 50 ms)
