Function

CenFcd21 manages central information used for automatic functions in a facade segment and distributes it either to subordinate application functions (AF) CenFcdShd11 or directly to the connected rooms.

A facade segment has windows with the same:

  • azimuth and slope
  • shading product
  • sun, e.g., shadows from above
  • shadows from the horizon and neighboring buildings
    This is relevant if you do not use the option "Anti-glare protection calculation with interface for annual shading calculation".

Solar angle

The solar angle on the facade (Relative solar azimuth angle, Relative solar elevation angle) is calculated from the geographic location, facade orientation, and date and time. It determines whether the sun is shining on the facade sector.

The following values are provided as BACnet objects for monitoring:

  • Relative solar azimuth angle (SolAzRel)
  • Relative solar elevation angle (SolEtRel)
  • Direct solar radiation on facade (SolRdDir)

Optionally, information from the annual shading computer can be used to account for shadows from buildings and the horizon.

Limits to direct solar radiation

The following factors limit radiation through windows:

  • The wall thickness (angle range left and right). You can configure this using SolAglLeft and SolAglRight.
  • The wall thickness above balconies. You can configure this with SolAglUp.
  • The horizon. You can configure this with SolAglDwn.

Delay after direct solar radiation

Glare does not stop immediately when the sun stops shining on the facade. Glare depends on solar elevation and on local conditions, e.g., reflections, bright surfaces, rounded edges.

You can define a switch-off delay in the function, where the function calculates the value based on time and facade orientation:

  • DlyOffSolDirHi (at vertical solar radiation)
  • DlyOffSolDirLo (at horizontal solar radiation)

Blinds type (strategy)

You can configure the blinds type and its response.

Strategy

Description

1

Blinds, position number (P-position)

Control blinds with 4 different slat positions (P-positions). It defines a slat angle in the lowered position (height 100 %).

Positions must be predefined in the actuator. They do not need to be specifically set. This strategy is suitable for almost all standard blinds.

2

Blinds, angle

The strategy is used only if the actuator does not support position numbers. Control is by height (always 100 %) and the corresponding slat angle.

3

Blinds, height, and angle

Blinds are adjusted based on sunlight in height as well as slat angle. The strategy is suitable for special applications, e.g., for blinds that move from bottom to top.

This requires detailed planning at the object.

4

Awnings, height (shadow edge)

The awning height is controlled based on solar position, so that the shadow edge is always at the same location in the room.

This requires detailed planning at the object.

Delayed operation

There are three more group managers for delayed operation. The goal is to distribute the rooms to more group managers and make sure that not all rooms change their operating mode at the same time.

CenFcd21 is part of the building scope and can be used in two ways:

  • For commanding of subordinate central functions in the floor scope
  • For commanding of subordinate room controllers in the room scope

The following diagram provides an overview of the two ways.
