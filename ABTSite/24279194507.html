[CnsmPrd21] Consumption per period

Program block, name / node subtype: [CnsmPrd21] Consumption per period

Library hierarchy: Universal functions

Family: CnsmPrd

Configurator

Program blocks are stored in the library without I/Os. Use the auto-create and auto-connect functions to create I/Os and/or to connect them to the interface blocks, e.g., R_A, CMD_B. Alternatively, take the I/Os from the folder containing the predefined I/Os in the library and/or from the plant and connect them to the interface blocks.

Function

CnsmPrd21 reads the truncated present value (property ID 4432) of an accumulator object and calculates the difference between the current consumption since the beginning of the period and the saved consumption from the last period. At the end of the current period, CnsmPrd21 saves it and displays it as a consumption from the last period.

The values for the current consumption from the present period and from the last period are saved persistently.

CnsmPrd21 can be used as an energy meter because it reads the value from an accumulator object and displays it as a real value with an analog calculated value.

A period is not a "moving" period. A period is a static period that is derived from the date and time:
A period of one hour is, e.g., between 06:00 and 07:00.
A period of one day is between 00:00 and 24:00.
A period of 15 minutes is between 06:00 and 06:15, then between 06:15 and 06:30, etc.

This program block has the following optional functions:

Option: Consumption per 15 minutes

It calculates the difference between the current consumption since the beginning of the period and the saved consumption from the last period. At the end of the current period, it saves it and displays it as a consumption from the last period (last 15 minutes).

Option: Consumption per hour

It calculates the difference between the current consumption since the beginning of the period and the saved consumption from the last period. At the end of the current period, it saves it and displays it as a consumption from the last period (last hour).

Option: Consumption per day

It calculates the difference between the current consumption since the beginning of the period and the saved consumption from the last period. At the end of the current period, it saves it and displays it as a consumption from the last period (last day).

Option: Consumption per week

It calculates the difference between the current consumption since the beginning of the period and the saved consumption from the last period. At the end of the current period, it saves it and displays it as a consumption from the last period (last week).

Option: Consumption per month

It calculates the difference between the current consumption since the beginning of the period and the saved consumption from the last period. At the end of the current period, it saves it and displays it as a consumption from the last period (last month).

Option: Consumption per year

It calculates the difference between the current consumption since the beginning of the period and the saved consumption from the last period. At the end of the current period, it saves it and displays it as a consumption from the last period (last year).
