[CmdSeqMon21] Command sequence monitoring

Program block, name / node subtype: [CmdSeqMon21] Command sequence monitoring

Library hierarchy: Ventilation and air conditioning functions

Family: CmdSeqMon

Project tree

Diagram

Configurator

Program blocks are stored in the library without I/Os. Use the auto-create and auto-connect functions to create I/Os and/or to connect them to the interface blocks, e.g., R_A, CMD_B. Alternatively, take the I/Os from the folder containing the predefined I/Os in the library and/or from the plant and connect them to the interface blocks.

Function

Monitors the success of the start-up process and the state of the dampers and fans for the running plant.

A parameter enables and disables the supervision.

This program block has the following optional functions:

Option: Fans and dampers monitoring

When the plant is running and the startup sequence has been completed (function block CMDSEQ_B is no longer "In Progress"), the fans (must be "On") and the dampers (must be "Opened") are monitored. If a fan stops or a damper does not open, an alarm and a signal that stop the plant are generated.

Option: Number of start attempts monitoring

In case of a high number of start attempts (parameterizable, e.g., 4 attempts) within a short period (parameterizable, e.g., 15 minutes), an alarm and a signal that stop the plant are generated.

Option: Startup duration monitoring

If the start-up process takes too long (parameterizable, e.g., 10 minutes), an alarm and a signal that stop the plant are generated. A possible reason for this could be a setting for an aggregate in the function block CMDSEQ_B for "Waiting for feedback" and a missing operating state signal from this aggregate and no internal alarm for missing feedback in it that would also stop the plant.
