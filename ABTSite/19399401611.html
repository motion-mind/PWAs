Mass commanding

The Mass commanding task sets parameters on multiple devices. For example, adjust all rooms to the economy state.

NOTICE

Incorrect settings are written to the automation station

The action is irreversible once the parameters for multiple devices are set.

  1. Select only automation stations affected by the change.

Set parameters for multiple devices

  1. The automation stations are online.
  2. An application template is available in the project; application templates are based on the selected application type, for example, VAV041_H.
  3. The function can only be applied to application types for related devices such as DXR1, DXR2, Intelligent Valve, and so on.
  1. Go to Startup.
  2. Open the Mass Commanding task.
  3. From drop-down menus, select an application template to view favorites.
  4. In Favorites pane, select a predefined or own favorite.
  5. In Details pane, select a device or functionality.
  6. In Parameters pane, change the parameters.
  7. Connect to your network device.
    Connect and disconnect
  8. Click Select devices & command above the Parameters pane.
  9. In the new window, select all the devices for the parameter changes.
  10. Confirm with Command and OK.
  1. Parameters are set across all selected devices.

Undo the last commanded value using mass commanding

The Undo Last Command in Mass Commanding enables the user to cancel the previous commands set under the Mass Commanding section.

  1. Perform steps given in Mass commanding.
  1. Open the Mass Commanding task again.
  2. To change some parameters, click Select devices & command above the Parameters pane.
  3. In the new window, select all the devices for the parameter changes.
  4. Confirm with Command and OK.
  5. Click Undo Last Command.
  1. The last commanded value is cancelled.
  2. "Undo mass command executed successfully. The commanded values are restored to last values." message is displayed in the Log viewer and status bar.

The Undo Last Command button is disabled when you are offline, not commanded any object, or switched the template after commanding.
