Modifying a configured HVAC module

Changing the HVAC module (always based on the library) with the Function configurator can include the following options:

  • Update existing function with the transfer of the engineered parameters (as far as possible)
  • Update existing function with default parameters
  • Replace existing function with a new compatible function

 

1. Replace an existing HVAC module

  1. The selected plant is created with nested modules.
  1. Select the module to be changed.
  2. Right-click the module and select Function configurator. Alternatively, right-click the empty space in the chart and click Function configurator.
  3. The Function configurator dialog box opens.
  4. Open the module list box. The module name is displayed with (engineered data).
    This means, that the module has been adjusted and no longer matches the original module from the library. Therefore, deleted options cannot be reactivated.
  5. Select the new required function:
    • Module (reset to defaults from library) (library) - the CFC block and BACnet value configuration will be reset and replaced with the defaults from the library.
    • Compatible module from the library (library).
  6. Clear the unused function from the selected module.

    Note: Assigned I/O data points are removed when a function is deselected.
  7. Adapt the functionality according to the system specification.
  8. Click OK.
  1. The modules are replaced.
  2. The deselected options are removed.
  3. The chart connections are updated.

I /O data points must be reassigned according to the new functionality.

2. Reassign I/O data points

When replacing a module, program data are copied from the library and existing I/O data points are reused as they fit. Additional I/O data points might be copied from the library, but they should be checked and completed manually.

  1. In the Project tree, go to the BACnet folder.
  2. Open the folder structure of the previous module that was replaced.
  3. Select the new module in the folder structure and verify the data points in the respective module in the Programming editor.
  4. Add missing I/O data points from the library to the control program as needed.
  5. Delete existing I/Os that are not required in the new program block.
  6. The hardware data are replaced.
