Auto-connect

The auto-connect function automatically connects items, typically data points with application functions, that have the same base name and extension and that are within the auto-connection scope.

If there are several devices or functions in the same segment, the auto-connect function differentiates between them by the base name extension. The base name extension can be a number or text.

The auto-connect function is triggered automatically.

Copying an object triggers the auto-connection. Moving an object keeps the references to objects. When moving objects, check assignments of I/Os and group members after moving the objects.

Base name and extension

Define or verify the base name and extension as soon as an item is added to avoid unwanted connections. Changing the base name is likely to create problems, such as missing connections or unwanted connections.

Auto-connection scope

The scope of the auto-connect function depends on the object for which the function is started.

If the auto-connect function is started for a room, the function considers all objects in the room. Objects in the room segments and lighting emergency zones assigned to the room are not considered.

If the auto-connect function is started for a room segment or a lighting emergency zone, the function considers all objects in the room segment or the lighting emergency zone.

If the auto-connect function is started for a central function area object, the function considers all objects in the central function area object.
