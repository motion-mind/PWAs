Extraction arm, pressure sensor and flow coefficient (ExtrcEqp13)

This application function monitors a pressure sensor on an analog input object (ExtrcDiffP) and calculates the air flow through the extraction arm using a data model with a constant box factor, without a separate factor for duct area.

The AF works with or without a binary input (ExtrcSpSel) to select the air flow setpoint. If a binary input is not connected, the setpoint will always equal ExtrcSp1. If a binary input is connected, the setpoint will equal ExtrcSp1 when the BI is active, and it will equal ExtrcSp2 when the BI is inactive.

The AF functions as described above. The main output signals are calculated values for Extraction present setpoint and Extraction air volume flow.

Main features:

  • Analog input pressure sensor monitor
  • Setpoints for extract airflow, configurable
  • Failure mode (configurable) for air volume flow sensor
  • Trend for extraction present setpoint
  • Trend for extraction air volume flow
  • Event enrollment for Monitoring for extraction air volume flow

Hysteresis for differential pressure. Related to the Switch-on point for differential pressure (SwiOnPtDiffP). Once the system reaches the switch-on point, the differential pressure sensor returns a non-zero value. After differential pressure passes above SwiOnPtDiff, the HysDiffP parameter sets how far the pressure measurement can fall below the switch-on point (SwiOnPtDiff), and still return a non-zero value. Time constant for air volume flow

The time constant for the attenuation filter on the air volume flow sensor. Larger values help to stabilize fluctuating air volume flow sensor values. Larger values for this constant also cause the values reported by the air volume flow sensor to be more heavily weighted toward measurements in the past. Leaving at 0 disables the filter.

Switch tolerance for tracking method to air volume flow

Sets an acceptable level of difference (percentage) between the relative airflow setpoint and current relative airflow.

By default, the system tracking method follows the airflow setpoint. But if current relative airflow deviates from the relative setpoint by a percentage amount greater than what is set by the SwiTolTckMthd parameter, the system switches from tracking to the setpoint to tracking to the actual airflow.

Setting the tolerance level (SwiTolTckMthd) to 100% will not always prevent tracking from switching from setpoint to airflow.

SwiTolTckMthd

5 [%]
