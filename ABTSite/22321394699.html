M-bus

The M-bus (Meter Bus) was designed to enable remote reading of utility meters, for example, heat cost allocation devices, heat-, water-, gas- meters and with some restrictions also electricity meters and heating controllers. The main application purpose of M-bus systems is to read out a complete and consistent set of metering data from the connected meters for billing and visualization/diagnostics purposes. The M-bus manager data acquisition acts as client and the subordinates act as servers to exchange messages in a request-response messaging pattern.

The M-bus manager implementation of the PXC4/5/7 device will be restricted to few basic and common protocol features. The PXC4/5/7 device will not make use of all other special features, options and variants of the M-bus specification, since the support and interpretation of these protocol mechanisms are manufacturer specific.
More information can be found in the M-bus expert documentation A6V16039082 on SID.

The M-bus task allows you to create a network topology and integrate networked field devices for measurement. You can add networks, devices and data points.

 (Alarming)

A checkbox allows to enable/disable the intrinsic alarm extension of the BACnet object.

Select to activate the alarming function for the object.
In the filter field, select True to filter for objects with activated alarming function.

Note: If several alarms are triggered by the same object (performs an alarm shower), only one alarm is sent to the corresponding recipients. This ensures that the user does not receive and have to acknowledge multiple alarms.

 (Trending)

A checkbox allows to create/delete a trend log for an IO. Created trend log is automatically linked to the current value of the IO.

Select to activate the trending function for the object.
In the filter field, select True to filter for objects with activated trending function.

Name

Data point object Name. Name is shown from plant level.
