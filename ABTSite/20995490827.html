Limitations and rules

Change object instance numbers in the csv file (for example, by Excel):

  • Only the object instance numbers are changeable. All other changes (for example, name, descriptions, types or adding additional objects) will be ignored ABT Site.
  • ABT Site checks the uniqueness or format the BACnet object instance numbers before import. Faulty instance numbers (for example, duplicated numbers or wrong format) file will block the import. The user gets a generic error message in this case. Use Excel to find the duplication, for more information see https://trumpexcel.com/find-and-remove-duplicates-in-excel/.
    Select Home > Conditional Formatting > Highlights Cells Rules > Duplicate Values.
  • After instance number changes (even for 1 object only) ABT Site force full load - no delta load (over Play/Pause) possible.
  • After instance number changes, Desigo CC and Desigo Control Point must re-learn /engineer from scratch. If that engineering is not started yet, in that case it has not impact for Desigo CC and Desigo Control Point.

The user is responsible to insure the uniqueness of the instance numbers => each object type counts up separately from 0 (zero).

Workflow

Create a data export

  1. Go to Building > Building structure.
  2. Select a device or a hierarchy object.
  3. Right-click and select Create export.
  4. The Create export dialog box opens.
  5. Select the Scope of content to export.
  6. Select the BACnet object instance numbers (PXC4/5/7 only) option.
    • (Optional) Enter a comment.
  7. Click OK.
  8. The selected content is exported to the default directory for exports [drive]:\Users\Public\Documents\Siemens\DESIGO\Exports.
  9. The Create export dialog box displays allowing you to open the containing folder.
  10. Click Open folder.
  11. The file explorer opens.
  12. Extract the created zip.file.
  1. For each device is a csv.file created with the corresponding device name.

Modify BACnet object instance number

  1. Open the csv.file with Excel.
  2. Rename the BACnet object instance number (column E) as required.
    INFO:
    - Do not change names in the column object-instancobjecte, if the object-instance-modifiable contains a No.
    - Do not rename column names or delete the header row.
  3. Save the changes.
  4. Import the file back to ABT Site.
    Importing modules and I/Os

Additional rows created are skipped and not imported. Deleted rows remain unchanged in the project.
