Checking hub connection

A floor hub acts as a hub to its nodes as well as a node to the building hub. The connection states must therefore be checked in both directions.

Checking connection from the floor hub to the nodes

  1. The user is logged into a hub.
  1. Go to Object view > Device > Infrastructure.
  2. Select Network port for secure connect and then .
  3. The Network port for secure connect properties work area opens.
  4. Select the Hub function connection status field.
    Note: The number refers to the number of nodes that belong to this hub.
  5. The Hub function connection status work area opens.
    Note: The number refers to the number of entries below this node.
  6. Select one of the Hub function connection status row.
    Note: Unfortunately, there is no condition overview at this level. Therefore, all entries must be opened individually to find the error. With up to 100 nodes this can take some time.
  7. The extended Hub function connection status work area opens.
  8. Analyze the information displayed for further troubleshooting.

 

Checking connection from a floor hub to a building hub

In this case, the floor hub acts as a node that connects to the building hub.

  1. Select Network port for secure connect and then .
  2. The Network port for secure connect properties work area opens.
  3. In the field Hub connection status, check the connection status.
  1. If Connected to primary hub is displayed, everything is fine and does not require further diagnosis. If a different message is displayed, more information can be displayed in the fields Primary hub connection status and Failover hub connection status. The number <5> refers to the number of individual pieces of information available for this field.

 

Error analysis

  1. The Hub connection status does not show the Connected to primary hub information.
    Error list
  1. Select either Primary hub connection status or Failover hub connection status field.
  2. The Primary hub connection status work area opens.
  3. Analyze the information displayed for further troubleshooting.

The time entry in the Connect timestamp row shows the time of the last connection attempt to the primary hub.
