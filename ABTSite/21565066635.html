Data tampering detection

WARNING

Cyberattack

Risk of manipulated project data and re-engineering of the entire project data and IT infrastructure

  1. Only use official Siemens ABT Site installations.
  2. Open used ports only.
  3. Create regular project backups.
  4. Please refer to the Desigo cybersecurity guidelines (A6V13813955) for further information about hardening.
  5. To become more familiar with the Desigo cybersecurity, watch the cybersecurity video for PXC4/5/7 (on YouTube).

A cyberattack on ABT Site can cause serious damage to Siemens devices or the customer IT infrastructure. Therefore, ABT Site continuously checks whether the installed files have been manipulated. As soon as a potential manipulation of a file is detected, the user is informed with a dialog box.

Function

Description

Show folder

Opens the diagnostic program folder of the log file location [Installed drive]:\ProgramData\Siemens\Desigo\ABT\Diagnostic\AbtSignatureValidation_[Date and Time].

Example of an error message:

The following files with invalid signature have been detected: \SampleExt.dll; Signature: invalid; Issued for: ''; Issued by: ''; Signature is present, but could not be verified for unknown reasons: C:\Program Files\SampleComponent\SampleExt.dll

Cancel

ABT Site will close immediately. The project data will remain in its last persisted state. If you have unsaved changes, depending on your workflow, those changes may be lost. We recommend closing ABT Site and informing Siemens support or/and your IT department.

Continue

The user is aware of the scope of the information. This can be used if, for example, a new test file is required to run a function.

Hide validation...

If the program is continued with Continue:

the message will re-appear periodically for the duration of the session.

the message can be suppressed for the duration of the session.
