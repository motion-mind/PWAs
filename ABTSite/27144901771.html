Synchronizing of the calendar objects

In order for global calendar changes to be synchronized with the other devices during commissioning, a manual synchronization must be carried out.

Synchronize the calendar entries in the entire project.

  1. Connection to Web interface is established on the calendar manager device.
  2. The devices with subordinate calendars are online.
  1. Select Object view > Device > Infrastructure.
  2. Select Synchronize.
  1. Synchronization starts.

Status messages:

  • Ready-success: No synchronization is in progress. Either no synchronization has been triggered yet or last synchronization was completely successful
  • Ready-failure: No synchronization is in progress. Synchronization has been triggered before and at least one calendar could not be synchronized. A new synchronization must be triggered by selecting Synchronize.
  • Synchronize: Synchronization is currently in progress.

Verify the calendar manager entries

  1. Select Object view > Device > Infrastructure.
  2. Select Calendar manger and then .
  3. The Calendar manger properties dialog opens.
    - The Primary devices field shows how many primary calendars in the project are defined.
    - The Subordinates field shows how many subordinate calendars have a reference to the primary calendars.
    - The Exchange status shows how many changes are successfully synchronized.
  4. Select the Primary devices field for detail information.
  5. The Calendar manager properties sub-dialog opens.
  6. Select the Subordinates field for detail information.
  7. The Subordinates properties sub-dialog opens.
