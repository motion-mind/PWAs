Repartitioning of DXR2 devices with device version V6.4

The firmware of the latest device version (V7.0) is larger and can no longer be loaded into the existing DXR2 devices with old firmware. Therefore, all DXR2 devices must be repartitioned, so that the latest firmware can be used. This requires several steps and takes several minutes for each device. Most actions can be performed with mass operation, which significantly reduces the time required in a project.

For existing DXR2’s, there is no update to a higher firmware version (including upgrade of the device type version) needed, as long as they stay within the original scope (for example, do not need new features or functions or texts).

IP connection for DXR2.E devices

With an IP connection, the selected DXR2.E devices are processed in parallel. Thus, approximately 20 minutes are required for 20 devices. It is not recommended to select more than 20 devices.
Connecting via an Ethernet/IP network

MS/TP connection for DXR2.M devices

The DXR2.M devices for MS/TP require a multiple of the time compared to IP devices. This is the result of the slow MS/TP communication, the amount of data and the sequential processing of the devices. It is recommended to run the repartitioning task overnight or during the weekend. The process requires approximately 3.5 hours for one device. We recommend performing a test in the office before repartitioning a DXR2.M device.
Connecting via an Ethernet/IP network

USB connection for DXR2.M devices

Use this connection type instead of the MS/TP connection if you can connect the DXR2M device directly to the computer. The repartition process will then take approximately 15 minutes per device.
Connecting via USB port

For critical deployments (for example, server room), the procedure must be scheduled with the customer in advance.

When multiple devices are selected for download and/or upload, one or more devices may fail to load due to network conditions, such as excessive network traffic. In these cases, select the devices that failed to load and repeat the download/upload command.

1. Check the installed root file system version

  1. The DXR2 devices are connected and operational.
    Connecting to devices
  1. Go to Startup.
  2. Open the Configure and download task.
  3. Select the Engineered devices tab.
  4. In the Discovered devices tab, click Discover and select a Discovery filter(All or Locally connected device)from the drop-down list.
  5. Click OK.
  6. All locally connected devices on the selected network display in the Discovered devices table.
  7. In the Discovered devices tab, select a device.
  8. In the Details pane, select the System tab. This check must be performed manually for each device.
  1. The device information appears:
    - Device type: DXR2.x
    - Firmware version: Lower than Desigo firmware library version FW_V7.0_DXR2_V01.21.152… -> Repartition required
    - Root file system: Small -> Repartition required

2. Read back data

  1. The devices are connected.
    Connecting to devices
  2. The root file system is small.
  1. Go to Startup.
  2. Open the Configure and download task.
  3. Select the Engineered devices tab.
  4. Select a hierarchy folder (for example, Floor).
    Note: The maximum of multi-selected devices should not be more than 20 devices.
  5. Right-click the hierarchy folder and select Upload.
  6. The devices runtime instance data is uploaded to the project.
  7. Click the Log tab to check the upload status.

 

3. Repartition of the firmware memory

  1. The required repartition firmware is integrated into ABT Site and cannot be manually selected at later time.
  2. This step will take with:
    - IP connection for 20 devices about 15 - 30 minutes
    - MS/TP connection for 1 device about 3 hours and 30 minutes
    - USB connection for 1 device about 15 – 20 minutes
  1. Select the Engineered devices tab.
  2. Select a hierarchy folder (for example, Floor).
    Note:
    - The maximum number of multi-selected devices should not exceed 20.
    - DXR2 devices that already have the large root file system loaded will be skipped.
  3. Right-click the hierarchy folder and select Repartition device.
  4. A Confirmation dialog opens.
    Note:
    Carefully read the confirmation information before proceeding. If there is a power failure during the repartitioning period, the devices can be damaged and must be replaced.
  5. Click Yes.
  6. Click the Log tab to check the progress of loading the repartitioner.
  7. When the download is finished, the Repartitioning has started on device dialog opens.


    Important:
    - The devices are no longer registered in ABT Site for several minutes.
    - There are no flashing LEDs during the repartition period on the device itself.
    - The device appears to have died but is still active in the background.
    - Never interrupt the power supply.
  8. Click OK.
  9. (Optional USB): From this point on, ABT Site can be disconnected from the device.
    Notice: If the power supply is interrupted during this time, the device will be destroyed.
  10. (Optional USB): Connect at later time when LEDs are flashing again.
  11. Wait till the device is visible in the Discovered devices tab.
  1. In the Device type column, the device type changes from DXR2.E18-1 to DXR2.E18-1X. The X in the device type indicates that the root file system is changed to large root file system and is ready for the update of the new firmware.
  2. Note: The repartition firmware is not operational for the control program, a firmware update with a production firmware is needed (see next step).

 

4. Update firmware from library

  1. Update files Desigo firmware library (FW_V7.0_DXR2_V01.21.152… or higher) are available on the computer.
  1. Select the Discovered devices tab.
  2. In the Device type column, filter for the DXR2 devices.
  3. Select all DXR2 devices with an extension X in the Device type name.
  4. Right-click the selected devices and select Firmware.
  5. Select the Library tab or Browse to a local folder.
  6. Select the device type and the (Desigo firmware library version FW_V7.0_DXR2_V01.21.152… or higher) from the corresponding drop-down list.
    Update firmware from file
  7. (Optional) Click View Release Notes for more information about the selected firmware version.
  8. Click OK.
  9. The device firmware is updated. The update process might take a while. Wait for the device to restart.
  10. Click the Log tab to check the status of the firmware upload.

 

5. Full download

  1. Select the Engineered devices tab.
  2. Select a hierarchy folder (for example, Floor).
  3. Right-click the selected folder and select Load > Full (device may restart).
  4. The control program is loaded and operational.
  5. Click the Log tab to check the status of the full download.
  6. Check the functionality of the control program in the DXR2 devices.
