Scenario 2: Two BACnet networks on different Ethernet ports

BACnet networks are logically routed and physically separated. BACnet routing is only possible between the Ethernet ports: /IP on one Ethernet port and /IP on the WAN port.
Configure flexible ports

 

The WAN and LAN port has unencrypted BACnet/IP traffic and is not intended to use for external network access.

 

Separate BACnet/IP traffic on IT/OT networks

This topology supports:

  • Separating BACnet/IP traffic on individual Ethernet ports with individual MAC addresses.
  • Only the PXC4/5/7 devices are visible to IT network monitoring.
  • Only BACnet application data between PXC5/7 Ethernet ports (BACnet data broker).
