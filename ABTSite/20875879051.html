Disabling BACnet/IP communication

You can deactivate BACnet/IP communications for devices which have BACnet/SC enabled (if the BACnet/IP communication is truly no longer needed). When deactivating BACnet/IP the devices will communicate BACnet/SC only. Other communication channels like BACnet/IP over ethernet and DICP over IP-multicast will be deactivated.

There are two exceptions.

  1. Port 443 for HTTPS communication will remain open.
  2. The port for cloud communication will remain open.

You can close and deactivate the BACnet/IP communication for one device, or for all devices connected to a router (as a mass operation).

Disable BACnet/IP communication for one device (in the BACnet/SC topology)
  1. Go to Building.
  2. Open the BACnet/SC task.
  3. In the BACnet/SC topology table, select a device.
  4. In the column BACnet/IP, select Disabled.
  1. The BACnet/IP communication is disabled and the IP functions in the Device details are grayed out.
Disable BACnet/IP communication for one device (in the Device details)
  1. Go to Building.
  2. Open the BACnet/SC task.
  3. In the BACnet/SC topology table select a device.
  4. In Device details > BACnet/SC, select Disable BACnet/IP.
  1. The BACnet/IP communication is disabled and the IP functions in the Device details are grayed out.
    Details pane of the device version ≤ 1.5
Disable BACnet/IP communication for all devices connected to a hub
  1. Go to Building.
  2. Open the BACnet/SC task.
  3. In the BACnet/SC topology table, select a device which is used for BACnet routing.
  4. Right-click and select Close all BACnet/IP communication.
  1. The BACnet/IP communication is disabled for all devices connected to the hub.
