[Fdp30] Fire damper, open/close function & MI for feedback only

Program block, name / node subtype: [Fdp30] Fire damper, open/close function & MI for feedback only

Library hierarchy: Ventilation and air conditioning equipment

Family: Fdp

Project tree

Diagram

Configurator

Program blocks are stored in the library without I/Os. Use the auto-create and auto-connect functions to create I/Os and/or to connect them to the interface blocks, e.g., R_A, CMD_B. Alternatively, take the I/Os from the folder containing the predefined I/Os in the library and/or from the plant and connect them to the interface blocks.

Function

Monitors a fire damper with open/close function with mandatory feedback signal for the opened position and for the closed position and optional fault alarm.

This program block is mainly used in FdpGrp21 as an alternative to Fdp22.

The opening and closing time can be parameterized in the program.

This program block has the following optional functions:

Option: Fault (1xBI)

Reads a fault contact and generates an alarm.

This option is mainly used for dampers with actuators that are integrated over bus, e.g., Modbus.

The program logic reads the binary command, considers the opening and the closing time, compares the signals from the end switches of the actuator from the "Feedback" to the command and can generate two alarms (binary calculated values): "Position monitoring (Opened)" and "Position monitoring (Closed)". If "Position monitoring (Opened)" is in alarm, the air handling unit stops.
