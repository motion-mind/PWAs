Adapting custom state text for an event enrollment object

The description is valid for binary and multistate objects. When reusing customer state texts in different objects, make sure that the signal number has the same number of channels.

1. Define the number of signals used for this data point

  1. A binary or multistate object is available.
    Creating I/O data points
  1. Go to Building > Building structure.
  2. Right-click a device and select Go to engineering.
  3. Open the I/O configuration task.
  4. In the Plants navigation view, select the binary or multistate data point.
  5. In the Property view, select the Signal property.
  6. From the Signal drop-down list, select the number of channels for this data point (default is 3 channels).
  7. In the Number of states property, the number of used channels is displayed.

 

2. Define the custom state text

  1. In the Property view, select the State text property.
  2. Double-click in the State text field.
  3. The State text dialog opens.
  4. Enter the custom text for each channel.

    or select existing standard texts.
  5. Click OK.

A customized text can be reused (with the same number channels only) for other objects within the same device.

 

3. Create an event enrollment object

  1. Open the BACnet references task.
  2. Select the Projects tab.
  3. Select the device, and then the binary or multistate object.
  4. Right-click the binary or multistate object and select Create event enrollment object.
  5. Select the Overview tab.
  6. Select the created event enrollment.
  7. In the Property view, select the Event type property.
  8. From the Event type drop-down list, select the Change of state option.
  9. In the Property view, select the Event property states type property.
  10. From the Event property states type drop-down list, select the Unsigned value option.

 

4. Assign the custom state text to the alarm value

  1. In the Property view, select the Event parameters property.
  2. Double-click in the Event parameters field.
  3. The Event parameters dialog opens.
  4. From the Alarm values 1 drop-down list, select the alarm text.
  5. Click Add alarm values to extend the alarm functionality.
