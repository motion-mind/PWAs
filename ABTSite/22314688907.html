Designing systems with BACnet/SC

IT/OT collaboration is required to realize secure smart building networks. Building automation system integrators need to collaborate with IT professionals when deploying secure OT networks with BACnet/SC.

Even though it is technically possible to have both, a BACnet/IP network driver and a BACnet/SC network driver in the workstation, strategically, it makes the most sense to configure the workstation to only communicate BACnet/SC with the primary system controllers and leverage BACnet routing for non-BACnet/SC networks in the same project. Workstations and primary controllers should be the first priority for BACnet/SC integrations because they are more likely to be on shared networks with public access, such as enterprise IT networks or networks with an internet connection, and there is less pressure to secure networks deep inside the building for now.
