First time configuration of HVAC module

The Function configurator allows a highly flexible adaptation of a compatible HVAC module within a plant, for example, an energy recovery module:

  • Within a chart, exchangeable modules are marked "adjustable.
  • Only compatible variants are displayed in the Function configurator.

Behavior of the Undo function in the Function configurator:
- When switching between the Function configurator editor and an ABT Site editor, the respective work steps are deleted. This means, that Undo can no longer be carried out when returning to the Function configurator editor.
- A performed user action in the Function configurator editor is saved in the program as several performed steps. Depending on the application, several Undo steps must be carried out to achieve the original state.

 

1. Replace a module

  1. The selected plant is created with nested modules.
  1. In the Plants navigation view, select the module to be changed.
  2. Right-click the module and select Function configurator. Alternatively, right-click the empty space in the chart and click Function configurator.
  3. The Function configurator dialog box opens.
  4. Clear the unused function from the selected module
    or
    select an alternative in the list box.
  5. If a function is selected that contains other interchangeable modules, these are also displayed.
    Note: Assigned I/O data points are removed when a function is deselected.
  6. Adapt the functionality according to the system specification.
  7. Click OK.
    Note: Cleared options can no longer be activated at later time. If an option has to be activated again, the module must be selected again from the library.
  1. The modules are replaced.
  2. The deselected options are removed.
  3. The chart connections are updated.

I /O data points must be reassigned according to the new functionality.

 

2. Reassign I/O data points

When replacing a module, program data are copied from the library and existing I/O data points are reused as they fit. Additional I/O data points might be copied from the library, but they should be checked and completed manually.

  1. In the Project tree, go to the BACnet folder.
  2. Open the folder structure of the previous module that was replaced.
  3. Select the new module in the folder structure and verify the data points in the respective module in the Programming editor.
  4. Add missing I/O data points from the library to the control program as needed.
  5. Delete existing I/Os that are not required in the new program block.
  6. The hardware data are replaced.
