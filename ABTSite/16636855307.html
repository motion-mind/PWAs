Extended checks

The following additional checks are performed when you click the button Load data for extended checks.

IP address range

All IP addresses including subnet masks must be in the range defined in Settings > Address defaults and in Building > Building structure.

MS/TP address range

  • All MS/TP addresses must be in the range defined in the Settings > Address defaults and in Building > Building structure. No check is performed on devices using auto-addressing.
  • Max manager must be equal for all devices with the same network number.
  • Max manager must be greater than or equal to the highest MAC address on a network.
  • Baud rate must be equal for all devices with the same network number.

Room checks in the project

  • Room ID must be unique for all rooms.
  • RCtl and ROpUnCtl group managers must have the same group number as the room ID.
  • If the room has ROpUnCtl, the group member number of ROpUnCtl in the room segment must be identical to the group member number of the ROpUnCtl in the room.
  • If the room has no ROpUnCtl, the group member number of ROpUnCtl in the room segment must be invalid and OSV = true.
  • Rooms without an assigned room segment should be disabled (manual correct => "Disable room").

Central function checks in the project

  • Group numbers of group managers used for central functions must be unique for each category in the project.
  • If a group member or group managerr has an invalid group number, OSV must be true.
  • A group member must have a group number of an existing manager.

Number range checks in the project

  • The device instance numbers must be between Device instance number start and Device instance number end.
  • The room IDs must be between Group number start and Group number end.
  • The group manager numbers in central functions must be between Group number start and Group number end.

Unique room names

  • The room names including the building hierarchy must be unique on device level.
  • The room names including the building hierarchy must be unique on project level.

Unique room segment names

  • The room segment names including the building hierarchy must be unique on device level.
  • The room segment names including the building hierarchy must be unique on project level.

Unique central function object names

  • The central function names including the building hierarchy must be unique on device level.
  • The central function names including the building hierarchy must be unique on project level.

Check for building hierarchy references of room segments

All room segments have an external reference into the building structure. During engineering, this is set according to the external reference of the assigned room. In the online system, the room assignment can be changed and therefore the reference into the building structure might be wrong.

Check for orphaned I/O assignments

The subordinate lists for I/O assignments are checked against available referenced devices in the project. Any missing device is logged with a warning message.
