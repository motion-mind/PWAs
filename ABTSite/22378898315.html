3. Setting up the devices for BACnet/SC

Before a conversion to BACnet/SC can take place, each device must first have been set up with IP communication. A direct configuration to BACnet/SC is not possible.

 

1. Establish network connection

  1. In the Common toolbar, select the arrow next to the Connect icon.
  2. Select the Ethernet tab.
  3. Select the Target selection option IP.
  4. Click Connect.
  1. The IP communication is established and operational.

 

2. Upload of the device data

To ensure that no changed parameters are lost, an upload must be carried out first.
Note: If BACnet/SC is commissioned directly, this step can be skipped.

  1. A network connection is established .
  1. Go to Startup.
  2. Open the Configure and download task.
  3. In the Engineered devices area, select the device.
  4. Right-click the device and select Upload.
  1. The current device data are uploaded and stored in the project data.

Any parameter changes made between the upload of the device data and the clear device procedure will be lost.

 

3. Assign the device a second time for BACnet/SC communication

  1. A network connection is established .
  1. Go to Startup.
  2. Open the Configure and download task.
  3. In the Engineered devices area, select the device.
  4. Right-click the device and select Find matching device type.
  5. The devices are listed in the Discovered devices tab.
  6. Select the device and click Assign.
  1. The device type is assigned to the device.

 

4. Download data to the device

  1. A network connection is established .
  1. Go to Startup.
  2. Open the Configure and download task.
  3. In the Engineered devices area, select the device.
  4. Right-click the device and select Load > Full (device may restart).
    Note:
    - Each hub and failover hub must be loaded individually. Multi-selection is not allowed.
    - Multi-selection for nodes is supported.
  1. The current device data is downloaded.
  2. The device is BACnet/SC configured and operational.

Shared BACnet objects, for example, outside air temperature may not work until the referenced device is upgraded to a BACnet/SC configuration.

 

5. Disable the BACnet/IP communication

The BACnet/IP communication must be disabled for all devices to ensure a secure communication through BACnet/SC.

  1. In the BACnet/SC topology tab, select the devices.
  2. Right-click and select Close all BACnet/IP communication.
  3. Right-click the device and select Load > Full (device may restart).
    Note:
    - Each hub and failover hub must be loaded individually. Multi-selection is not allowed.
    - Multi-selection for nodes is supported.
  1. The current device data is downloaded.
  2. The BACnet/IP communication is disabled for all devices connected to the hub.
  3. The BACnet/SC communication is operational and save.

 

6. Disconnect the network and reconnect as BACnet/SC connection

  1. In the Common toolbar, select the arrow next to the Connect icon.
  2. The IP communication is interrupted.
  3. In the Common toolbar, select the arrow next to the Connect icon.
  4. Select the Ethernet tab.
  5. Select the Target selection option IP.
  6. Select the BACnet/SC check box.
  7. In the Hub address drop-down list, select the hub or failover hub.
  8. Click Connect.
  1. The BACnet/SC communication is established.
  2. The communication icon changed to .
