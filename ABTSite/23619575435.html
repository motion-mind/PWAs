Transition Floor hub, 2nd floor to BACnet/SC (BACnet/IP enabled - IP routing enabled)

Configuration ABT Site – BACnet/SC

Shows the BACnet/SC configuration in ABT Site. BACnet/IP is set to Disabled on devices AS_11 – Floor hub 1st floor, AS_12 – Floor node 1st floor, AS_21 Floor hub 2nd floor und AS_22 – Floor node 2nd floor, while the others are set to Enabled. In addition, BACnet/IP routing on device AS_21 – Floor hub 2nd floor is enabled BACnet/IP is disabled on the remaining devices. Disabling BACnet/IP on device Floor hub 2nd floor also disable BACnet/IP routing. Devices AS_21 Floor hub 2nd floor and AS_22 Floor node 2nd floor can no longer be reached via BACnet/IP.

 

Topology and data communication

Illustrates which devices can communicate data as well how the data is transmitted. Disabling BACnet/IP on device AS_11 – Floor hub 1st floor, AS_12 – Floor node 1st floor, and AS_22 – Floor node 2nd floor renders the devices and BACnet objects invisible from the point of view of BACnet/IP. However, activating BACnet/IP routing on device AS_21 – Floor hub 2nd floor renders controller AS_22 – Floor node 2nd floor visible again.

The devices can no longer be accessed via BACEye and the BACnet references from the XWP PX 100 controller on Floor hub 1st floor and Floor node 1st floor no longer operates (reliability). Referencing to device AS_22 – Floor node 2nd floor operates again, since the device can be reached using the routing function on device AS_21 – Floor hub 2nd floor.

 

The BACnet/SC connection from the building hub to the floor node illustrates a visual communication path to depict functionality only. Data is always transmitted via the floor hub!
