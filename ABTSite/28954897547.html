Adding and renaming a presence detector manager

  1. Go to Building > Building structure.
  2. Right-click a device and select Go to engineering.
  3. Open the KNX PL-Link task.
  4. In the KNX PL-Link area, click Add network. Info: Depending on the used device, up to 5 networks for Modbus, M-bus, KNX PL-Link are supported.
  5. Select the Library view on the right.
  6. From the Field devices > KNX PL-Link folder, drag a presence detector from the library to the KNX PL-Link bus network. The presence detector is added to the plant, marked as favorite . If required, it can be moved to another plant. Alternatively, drag the presence detector directly to any plant.
    Notes:
    - Up to 64 KNX-PL-Link devices (manager and subordinate devices) per PXC4/5/7 are supported.
    - Up to 10 KNX PL-Link devices without additional KNX PL-Link power supply are supported.
    - Depending on the Desigo PXC4/5/7 device version, only the corresponding presence detector library is available in the library folder.
  7. Right-click the device and select Rename.
  8. Enter the name for the device, for example, Room 103.
  1. The presence detector manager device is added and renamed.

Check the I/O address

  1. Select the device.
  2. Select the Properties view on the right.
  3. Select the BACnet view option.
  4. Select > Ctr to open the detail parameters.
  5. Check the I/O address, for example, PL-1:DL=4;MS=M; (M for manager device).
    Note: MS=M is only added to the address string if this presence detector has a subordinate (only then is the presence detector a manager).

 

Verify the device role of the presence detector devices

  • In the KNX PL-Link area, select KNX PL-Link bus.
  • In the Device role column, the device role of the presence detector is displayed.
    Note: A standard device does not have any subordinate devices.
